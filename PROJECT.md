# TEKITO プロジェクト詳細仕様

## 機能詳細

### 1. LessonPage（レッスンページ）

#### 機能概要
学習コンテンツと解説を提供するページ

#### コンテンツ形式
- **テキスト**: 基本的なテキストコンテンツ
- **動画**: YouTube埋め込み（別途アップロードした動画）
- **インタラクティブ**: アニメーションを使用した手を動かして学べる形式

#### 仕様
- **作成方法**: 手動作成
- **進捗管理**: 不要（即レスポンス重視）
- **課題**: インタラクティブ要素の実装方法

### 2. QuizPage（クイズページ）

#### 機能概要
4択問題で理解度を確認するページ

#### 仕様
- **作成方法**: 手動作成
- **関連性**: Lessonとは独立、相互リンクで接続
- **結果保存**: 不要（即レスポンス重視）
- **難易度**: 低

### 3. GeneratePage（PDFツール）

#### 機能概要
教育者向けの学習資料（問題集）生成機能

#### 仕様
- **対象ユーザー**: 教育者向け
- **生成方法**: テンプレートベース
- **入力方法**: 検討中（後述の提案を参照）
- **優先度**: 低（後回し）

## 技術仕様

### データ保存方法

**推奨: JSONファイル方式**

- `data/lessons.json` - レッスンデータ
- `data/quizzes.json` - クイズデータ
- GitHubにコミットして管理
- デプロイ時に一緒にデプロイされる
- 認証不要で実装が簡単

**代替案**:
- ローカルストレージ（localStorage）: ブラウザに保存、デバイス間で共有不可
- データベース（Firebase/Supabase）: 将来的に拡張が必要な場合

### 認証・ユーザー管理

- **認証**: 不要
- **ユーザー管理**: 不要

### 将来の機能

- **AIチャットボット**: 無料のAIサービスがあれば実装予定（後回し）

## 実装順序の提案

### 推奨順序: Quiz → Lesson → Generate

#### 1. QuizPageから始める（推奨）

**理由**:
- 難易度が最も低い
- MVPとして最適
- 早く実装でき、フィードバックを得やすい
- LessonPageのインタラクティブ要素の実装前に、基本的なページ構造を理解できる

#### 2. LessonPage

**理由**:
- ユーザーが最初に触る確率が高い
- QuizPageの実装で得た知見を活かせる
- インタラクティブ要素は段階的に実装（まずはテキストと動画から）

#### 3. GeneratePage

**理由**:
- 優先度が低い
- 教育者向けなので、学習者向け機能が完成してからでも良い

## 迷っている箇所への提案

### 1. GeneratePageの入力方法

#### 提案A: フォーム入力方式（推奨）
- 問題文、選択肢、正解をテキスト入力
- 複数問題を一括で入力可能
- シンプルで実装が容易
- 後で他の方式を追加可能

#### 提案B: ファイルアップロード方式
- CSV/JSON形式で問題データをアップロード
- 大量の問題を作成する教育者向け
- 実装がやや複雑

#### 提案C: ステップバイステップ方式
- 1問ずつ入力フォームで作成
- UIが分かりやすい
- 実装は中程度

**推奨**: 提案A（フォーム入力方式）から始める。後で提案BやCを追加可能。

### 2. LessonPageのインタラクティブ要素

#### 提案A: CSSアニメーション + React State
- シンプルなアニメーション（フェードイン、スライド、ホバー効果など）
- 実装が容易
- パフォーマンスが良い

#### 提案B: Framer Motion（推奨）
- 豊富なアニメーション機能
- 実装が比較的容易
- パフォーマンスも良好
- Reactとの統合がスムーズ

#### 提案C: カスタムCanvas/WebGL
- 高度なインタラクティブ要素が可能
- 実装が複雑
- パフォーマンスに注意が必要

**推奨**: 提案B（Framer Motion）から始める。必要に応じて提案Aで補完。高度な機能は後で追加。

## 実装の進め方

### Phase 1: QuizPageの実装（MVP）

1. 基本的な4択問題の表示
2. 選択肢のクリックで正解/不正解の表示
3. 次の問題への遷移
4. データはJSONファイルから読み込み
5. レスポンシブデザインの確認

### Phase 2: LessonPageの基本機能

1. テキストコンテンツの表示
2. YouTube動画の埋め込み
3. 基本的なレイアウト
4. レスポンシブデザインの確認

### Phase 3: LessonPageのインタラクティブ要素

1. Framer Motionを導入
2. シンプルなアニメーションから実装
3. 段階的に機能を拡張
4. パフォーマンステスト

### Phase 4: GeneratePageの実装

1. フォーム入力方式で問題集生成
2. PDF生成機能（react-pdfやjsPDFなどを検討）
3. テンプレートの実装
4. ダウンロード機能

      "category": "数学",

## デザイン原則

- **気軽さ**: 即レスポンスを重視
- **モバイルファースト**: スマホユーザーを最優先
- **シンプル**: 複雑な機能は避ける
- **直感的**: 迷わず使えるUI
